# 评分标准测试日志

## 1. 目的

本日志记录当前 OCR + 作文评分标准系统在不同标准版本下的真实回归结果，用于：

- 追踪标准系统的演化
- 评估规则调整是否有效
- 为后续继续校准提供依据

注意：

- 这些结果属于训练/校准阶段证据，不是生产系统对外输出。
- 对照基准为人工 `86题` 分数。

---

## 2. 测试对象

### 数据规模

- 样本数：50 篇作文
- 输入形态：OCR 文本评分（非图片直投）
- 对照来源：人工 `86题` 分数

### 测试目标

比较不同标准版本在以下指标上的表现：

- 平均偏差
- MAE
- RMSE
- Pearson
- 完全一致率
- 误差 `<=1 / <=2 / <=3`
- 档位一致率
- 0 分/无效作文识别能力

---

## 4. 版本间对比总结

### 4.1 Baseline -> v1.1

改善：
- MAE: `2.86 -> 2.78`
- RMSE: `4.22 -> 3.90`
- `<=1`: `40% -> 44%`
- `<=2`: `58% -> 60%`

退步：
- `<=3`: `76% -> 70%`
- 档位一致率：`36% -> 32%`
- 0 分命中：`5/5 -> 2/4`

判断：
- v1.1 更强调“有效规则单元”和“高分容错”
- 方向部分正确
- 但未能压住低分边界

### 4.2 v1.1 -> v2

改善：
- MAE: `2.78 -> 2.66`
- `<=1`: `44% -> 50%`
- `<=2`: `60% -> 62%`
- `<=3`: `70% -> 72%`
- 档位一致率：`32% -> 48%`
- 0 分命中：`2/4 -> 3/4`

基本持平：
- RMSE: `3.90 -> 3.91`
- Pearson: `0.876 -> 0.869`

判断：
- v2 明显优于 v1.1
- 门控 + 封顶 + 复核 的体系开始真正奏效

---

## 5. 当前最主要问题

即使在 v2 中，仍然存在明显低分高估：

- 李梦然：人工 0，AI 13
- 司裕然：人工 2，AI 14
- 徐铭浚：人工 1，AI 9
- 赵嘉润：人工 0，AI 7

这些样本共同表明：

- 有书信模板壳子
- 沾一点题意
- 但正文规则展开不成立
- 单句语义/结构不可恢复

因此，当前继续校准的核心仍是：

```text
0-5 分最低分边界
1-9 分 / 10-14 分进入条件
有效内容单元的“稳定可复述”边界
```

---

## 6. 当前结论

### 已被证明有效

- Framework / Task Spec 分层方向
- 门控层
- 封顶层
- 复核层
- 高分语言容错原则
- 有效内容单元作为核心抽象

### 仍在继续校准

- task_not_established 的硬触发边界
- 0/1/2/3/4/5 分最低分细分
- 模板壳子对结构感知的干扰
- OCR 差样本在极低分段的归类

---

## 7. 当前阶段定位

当前评分标准系统已经达到：

```text
结构成熟、规则初步稳定、具备准生产潜力，
但低分边界仍处于继续校准阶段。
```

可以认为其成熟度位于：

```text
Level 2.5 ~ Level 3
```

即：
- 已经超出试验性 prompt
- 标准体系基本成型
- 可以继续以真实样本做定向边界校准
