# 当前评分标准系统成熟度评估

## 一、评估目的

本文件用于回答当前这套作文评分标准系统在什么成熟阶段、哪些部分已经稳定、哪些部分仍需继续校准。

注意：

- 本评估面向**评分标准系统本身**，不是前端、OCR、智学网接入或自动化流程评估。
- 当前的人工分对照、误差报告、MAE/RMSE 等，只属于**训练/校准阶段证据**，不属于未来正式评分系统的对外输出。

---

## 二、当前系统成熟度结论

### 总体判断

```text
当前评分标准系统：已达到“结构成熟、规则初步稳定、可持续迭代”的阶段；
但还没有达到“可完全替代人工直接上线”的最终成熟度。
```

更细一点说：

- **架构成熟度：高**
- **标准表达成熟度：高**
- **题目抽象能力：中高**
- **低分边界稳定性：中**
- **高分容错稳定性：中高**
- **直接生产可用性：中**

也就是说，这套系统已经不是“试验性 prompt”，而是一套有：

- 通用评分框架
- 题目任务配置
- 门控/封顶/复核机制
- 真实数据反推与校准路径

的标准体系。

但它仍然保留了典型的训练期特征：

- 低分区仍有高估残留
- 极端边界样本仍需继续压实
- 不同题目迁移时还需要更多任务配置验证

---

## 三、已经成熟的部分

### 1. 标准体系分层已经成熟

当前已经形成：

```text
Framework（通用评分框架）
Task Spec（题目任务配置）
Prompt Template（执行投影层）
```

这是一个正确且稳定的长期结构。

成熟表现：

- Framework 不再绑定具体题目词
- Task Spec 承担题目要求
- Prompt 不再被视为唯一规则源

这意味着后面换作文题时，不需要推翻评分系统本体，只需新增任务配置。

### 2. 门控 + 封顶 + 复核 的方向已经成熟

当前已经明确：

- 不是直接给总分
- 先判无效作文 / task_not_established
- 再走分维度评分
- 再走 caps
- 最后输出复核建议

这一点和市面上更成熟的自动作文评分系统是同方向的，说明架构判断已经稳了。

### 3. 五维评分框架已经成熟

当前五维：

- 切题与主题鲜明度
- 内容完整性与要点覆盖
- 语言准确性
- 结构与逻辑
- 书写规范性与卷面可读性

这套维度本身已经能覆盖当前题目，也有较强迁移潜力。

### 4. “有效内容单元”这一抽象已经成熟

这是当前标准系统里最有价值的抽象之一。

从“关键词命中”升级到“有效内容单元”，使得系统开始具备：

- 防低分高估
- 防模板灌水
- 防纯长度偏置
- 更强的任务完成度判断能力

这层抽象已经可以视为稳定核心。

### 5. 高分容错原则已经成熟

通过多轮样本验证，我们已经比较确定：

```text
高分作文不应因为拼写/语法/OCR 粘连错误多，就机械压低。
```

这条原则已经被真实样本反复验证，是当前系统中较稳定的一部分。

---

## 四、还处于继续校准阶段的部分

### 1. 0-14 分区间的边界稳定性

这是当前最明显仍未完全收敛的部分。

尤其是：

- `0分`
- `1-4分`
- `5-9分`
- `10-14分`

之间的细边界，仍然会受：

- OCR 破碎程度
- 模型对“似乎相关”的宽容度
- 模糊句意的恢复方式

影响。

### 2. `task_not_established` 的触发条件仍需继续压实

方向已经对，但还没有完全定死：

- 什么程度算“任务未成立”
- 只沾题但正文不成立，什么时候该直接压进 1-9
- 双侧展开不稳时，到底是否允许进入 10-14

目前这部分属于**半稳定规则**。

### 3. “有效内容单元”的计数边界仍需继续打磨

当前已经明确：

- 模糊可猜 ≠ 有效内容单元
- 关键词碎片 ≠ 有效内容单元

但还没有完全稳定的是：

- 多少信息才能叫“稳定可复述”
- OCR 粘连到什么程度时，仍可视作有效单元
- 1 条 vs 2 条 vs 3 条边界如何更一致

这是目前最值得继续精修的规则之一。

### 4. 低分样本中的“模板壳子”影响仍需继续压缩

当前已经有原则：

- Dear Eric / Yours, Mary 不等于正文成立

但在模型实际判分时，模板结构仍有时会抬高：

- 结构分
- 任务完成感知

这一点在极低分区仍需继续校准。

### 5. 书写/卷面维度在纯文本模式下的作用边界

这部分方向清楚：

- 文本模式弱判断
- 不让书写项主导总分

但实际量化边界仍在校准中：

- OCR 差时到底算语言问题还是可读性问题
- 书写项在低分区到底扣多少更合理

---

## 五、当前已具备的“正式系统潜力”

如果从“以后能否成为正式评分系统”的角度看，当前这套标准系统已经具备以下潜力：

### 已经具备

- 稳定的业务边界
- 稳定的分层架构
- 统一的输出 schema 思路
- 训练期可持续校准机制
- 对题目变化的抽象能力

### 尚不具备

- 低分区的最终稳定性
- 跨更多题目的泛化验证
- 长期生产环境下的边界样本完整库

因此，当前它更适合被表述为：

```text
已进入“准生产标准系统”阶段，
但仍需针对低分边界继续做有限轮次校准。
```

---

## 六、成熟度分级评估

我建议把当前系统定义为：

### Level 0：纯 prompt 试验
- 无统一 rubric
- 无稳定输出结构
- 无分层规则

### Level 1：结构化评分试验
- 有 rubric
- 有输出 schema
- 仍主要依赖 prompt 微调

### Level 2：标准系统初成型
- 有 Framework / Task Spec 分层
- 有门控 / 封顶 / 复核机制
- 有真实数据校准
- 但低分区边界未收敛

### Level 3：准生产标准系统
- 大部分规则稳定
- 只剩局部边界继续校准
- 可用于受控生产试运行

### Level 4：生产稳定系统
- 多题验证充分
- 低分/高分边界稳定
- 可作为正式自动评分主系统

#### 当前定位

```text
当前系统位于 Level 2.5 ~ Level 3 之间。
```

理由：

- 已经超出“试验 prompt”阶段
- 结构化标准体系已成型
- v2 已明显优于 v1.1
- 但低分区边界仍然没有完全收敛

---

## 七、当前最稳定 vs 最不稳定的部分

### 最稳定的部分

1. Framework / Task Spec 的抽象方向
2. 五维评分框架
3. 高分容错原则
4. 门控 / caps / review 的整体结构
5. 题目任务槽位思路

### 最不稳定的部分

1. 0-5 分最低分段的边界
2. task_not_established 的严格触发口径
3. 有效内容单元的边界计数
4. 模板壳子对结构感知的干扰
5. OCR 差样本在低分区的归类

---

## 八、是否已经奏效

答案是：

```text
已经奏效，但还没有完全收敛。
```

这里的“奏效”不是主观判断，而是已经在真实重跑中体现：

- v2 比 v1.1 的 MAE 更低
- <=1/2/3 分误差比例更好
- 档位一致率显著提升
- 0 分识别有改善

所以：

- 这套标准不是无效的
- 方向不是错的
- 只是还没到“完全替代人工直接上线”的成熟度

---

## 九、当前建议

### 1. 可以停止继续抽象结构

Framework、Task Spec、Prompt 的关系已经清楚，不需要再大改架构。

### 2. 可以开始围绕低分区做有限轮次精修

重点只盯：

- `0-5分`
- `1-9分 / 10-14分` 边界
- `task_not_established`
- 有效内容单元稳定可复述原则

### 3. 在更多题目出现前，不建议再大改通用框架

现在更适合的节奏是：

```text
框架冻结
任务配置按题扩展
边界规则按题校准
```

---

## 十、一句话总结

当前评分标准系统已经从“实验性 prompt”成长为：

```text
结构成熟、规则初步稳定、具备准生产潜力，
但低分边界仍处于继续校准阶段的标准系统。
```
