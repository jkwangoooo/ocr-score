# 英语作文评分通用框架（Framework v1）

## 1. 设计目标

本框架是**题目无关**的英语作文评分业务母版，用来承载：

- 可评分性判断
- 任务完成度判断
- 分维度评分
- 上限控制（封顶）
- 复核分流
- 结构化输出

它不包含具体题目的关键词、题干、内容点，也不依赖：

- OCR 实现
- 页面接入
- 本地文件夹结构
- 智学网/其他平台 API

题目相关内容应由单独的 **Task Spec** 提供。

---

## 2. 总体架构

评分系统分为四层：

### Layer 1: Gating（门控层）
先判断作文是否进入正常评分。

### Layer 2: Scoring（核心评分层）
仅对通过门控的作文，按维度评分。

### Layer 3: Caps（封顶层）
当任务完成不充分、内容单元不足、主题侧不完整时，限制总分上限。

### Layer 4: Review（复核层）
输出人工复核建议与风险标签。

---

## 3. 通用领域对象

### 3.1 Framework

Framework 是题目无关的评分规则母版。

建议包含：

- 评分维度定义
- 门控规则类型
- 封顶规则类型
- 复核规则类型
- 输出 schema 约束

### 3.2 Task Spec

Task Spec 是某个具体题目的配置，不改变框架本身，只填充：

- 题目说明
- 任务槽位（content slots）
- 本题高分要求
- 本题跑题特征
- 本题特定无效作文样态

### 3.3 Effective Content Unit

通用框架中不再写“规则单元”，统一抽象为：

```text
effective_content_unit
```

定义：

一段内容只有同时满足以下条件，才算有效内容单元：

1. 能被识别为与题目任务直接相关的一条具体信息单元
2. 能归属到某个任务槽位
3. 不是关键词碎片、空泛套话、模板化填充
4. 能被稳定复述为一条清楚意思

这一定义对任何作文题都适用。

---

## 4. 通用门控层（Gating）

### G0. Invalid Essay（无效作文）

命中任一项，直接：

- `invalid_essay=true`
- `score=0`
- `band=0`

#### G0-1 完全偏题
主体内容与题目任务无关。

#### G0-2 疑似套作/抄材料且不贴题
语言完整但任务不相关。

#### G0-3 只有模板无正文
只保留开头/结尾模板，无有效正文。

#### G0-4 正文不可判读
正文几乎无法恢复有效内容。

### G1. Task Not Established（任务未成立）

不直接 0 分，但进入低分锁定区。

适用情形：

- 只出现题目相关关键词
- 但没有形成稳定、清楚、可复述的任务内容展开
- 无法确认多个任务槽位是否成立
- 只能模糊猜到极少量有效内容单元

门控输出建议：

```json
{
  "gating": {
    "invalid_essay": false,
    "task_established": false,
    "flags": ["task_not_established"]
  }
}
```

## 5. 通用评分维度（Scoring）

### A. Task Relevance & Completion（任务相关性与完成度）

考察：

- 是否回应题目核心
- 是否覆盖必要任务槽位
- 是否主题鲜明

### B. Content Development（内容展开与完整性）

考察：

- 有效内容单元数量
- 内容是否展开
- 是否有必要的例子/说明/总结

### C. Language Accuracy（语言准确性）

考察：

- 词汇
- 语法
- 时态
- 拼写
- 句法

注意：

- 语言错误不能机械压分
- 只有当语言错误**破坏主要理解**时，才重扣

### D. Structure & Logic（结构与逻辑）

考察：

- 结构是否成立
- 段落组织是否清楚
- 句间逻辑是否可跟随
- 题目要求的文体格式是否成立（如书信、议论文等）

### E. Presentation / Readability（卷面与可读性）

考察：

- 书写/卷面是否影响识别
- 若只提供文本，此维度只能弱判断

---

## 6. 通用封顶层（Caps）

封顶规则不是具体题目词，而是题目无关的业务模式。

### C1. Missing Required Task Slots Cap

若未覆盖所有必要任务槽位：

- 总分封顶

### C2. Too Few Effective Content Units Cap

若有效内容单元数量不足：

- 总分封顶

### C3. Keyword-only Relevance Cap

若只有弱相关词，不形成清楚内容展开：

- 总分封顶

### C4. Template-like Low-density Cap

若模板感强、有效信息密度低：

- 总分封顶

### C5. Single-side / Partial-task Cap

若任务要求多侧/多点回答，但只完成一部分：

- 总分封顶

---

## 7. 通用低分区间定义

### 0 分

- 偏题
- 套作/抄材料且不贴题
- 只有模板
- 正文不可判读

### 1-4 分

- 有极少量相关痕迹
- 但无法形成有效内容展开
- 有效内容单元几乎不存在

### 5-9 分

- 任务弱相关
- 仅能识别少量有效内容单元
- 远不足以构成完整回答

### 10-14 分

只有在以下条件同时成立时才允许进入：

1. 至少能识别若干有效内容单元
2. 不是纯关键词沾题
3. 内容有基本展开
4. 主要文本仍可跟随

否则应压回 1-9。

---

## 8. 通用高分区间定义

高分区不以语言完美为前提，而以：

- 任务槽位覆盖完整
- 有效内容单元充足
- 结构成立
- 主要意思清楚

为前提。

因此：

- 高分可容忍较多拼写、语法、OCR 级错误
- 只要这些错误不破坏主要理解，就不应机械压到中低档

---

## 9. 通用复核层（Review）

### 必复核

- `invalid_essay=true`
- 命中封顶规则
- 极低分（0-5）

### 建议复核

- 文本过短
- 可读性差
- 任务槽位覆盖边界不稳
- 有效内容单元计数不稳
- 疑似套作/抄材料但证据不硬

---

## 10. 通用输出结构

建议统一输出：

```json
{
  "framework_id": "essay_scoring_framework_v1",
  "task_spec_id": "...",
  "invalid_essay": false,
  "invalid_reason": "",
  "gating": {
    "task_established": true,
    "flags": []
  },
  "score": 0,
  "band": "",
  "dimension_scores": {},
  "caps_applied": [],
  "reasons": [],
  "comment": "",
  "review_required": false,
  "review_reasons": [],
  "readability_notes": ""
}
```

---

## 11. 框架与任务配置的分工

### Framework 负责

- 什么叫无效作文
- 什么叫任务未成立
- 什么叫有效内容单元
- 什么叫封顶
- 什么叫复核

### Task Spec 负责

- 这道题有哪些任务槽位
- 哪些槽位是必答的
- 本题高分样态
- 本题常见跑题样态
- 本题文体要求

---

## 12. 结论

这份 Framework 的目标是：

```text
让评分业务本体稳定下来，
以后换任何作文题，只换 Task Spec，不改评分框架。
```
